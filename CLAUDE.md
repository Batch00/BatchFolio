# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Behavioral Guidelines

1. **Think Before Coding** - State assumptions explicitly before implementing. If multiple interpretations exist, present them. If a simpler approach exists, say so. If something is unclear, stop and ask.

2. **Simplicity First** - Write the minimum code that solves the problem. No features beyond what was asked. No abstractions for single-use code. No speculative flexibility or configurability. If you write 200 lines and it could be 50, rewrite it.

3. **Surgical Changes** - Touch only what you must. Do not improve adjacent code that was not asked about. Do not refactor things that are not broken. Match existing style even if you would do it differently. If you notice unrelated dead code, mention it but do not delete it. Remove imports/variables/functions that YOUR changes made unused - do not remove pre-existing dead code unless asked. Every changed line should trace directly to the user's request.

4. **Goal-Driven Execution** - Transform tasks into verifiable goals. For multi-step tasks, state a brief plan with verify steps. Strong success criteria let you loop independently. Clarifying questions should come before implementation.

## Architecture

Next.js 15 App Router, JavaScript (no TypeScript), Tailwind CSS, shadcn/ui components, Supabase auth + database, Recharts for charts, SimpleFIN for bank sync, Polygon.io for chart data, dnd-kit for drag-and-drop. Deployed on Vercel at batchfolio.batch-apps.com.

**Route groups:**
- `app/(auth)/` - login and signup pages, no nav
- `app/(dashboard)/` - single protected page at `/`, wrapped in `layout.jsx`
- `app/api/stock/` - server-side stock data proxy routes
- `app/api/simplefin/` - SimpleFIN sync routes
- `app/api/admin/` - invite management routes

**After login, the entire app lives at `/`.** There is no page routing between dashboard views. A sticky 48px top bar holds the logo, tab navigation (OVERVIEW / ACCOUNTS / PORTFOLIO / WATCHLIST), persistent net worth display, a LIVE indicator dot, +ADD button, and settings icon. Tab content is swapped in place below the top bar - not via URL routing.

**Auth:** Middleware at `middleware.js` handles all redirects. Unauthenticated users hitting dashboard routes go to `/login`; authenticated users hitting auth pages go to `/`. All dashboard components are `'use client'` and use `@/lib/supabase` (browser client) for data operations.

**Stock data is server-side only.** All Finnhub and Polygon calls must go through `/api/stock/*` routes. Never call these APIs from client components. `FINNHUB_API_KEY` and `POLYGON_API_KEY` are never exposed to the browser.

**Supabase clients:**
- `lib/supabase.js` - browser client (`createBrowserClient` from `@supabase/ssr`), initialized with `{ db: { schema: 'batchfolio' } }`, used in client components
- `lib/supabase-server.js` - server client (`createServerClient`), used in server components and middleware
- RLS is enabled on all tables; queries automatically scope to `auth.uid()`
- All tables are in the `batchfolio` schema, not `public`

**shadcn components** live in `components/ui/`. All components are manually written (not generated by CLI) to match the dark theme. Do not override their dark styling.

**Top-bar tab components** in `components/dashboard/`:
- `OverviewTab.jsx` - hero row (3 cols) + bottom widget grid (3 cols)
- `AccountsTab.jsx` - expandable account cards with holdings tables
- `PortfolioTab.jsx` - sortable full holdings table + allocation donut
- `WatchlistTab.jsx` - full watchlist table with fundamentals

**Overview widgets** in `components/dashboard/`:
- `NetWorthWidget.jsx` - big net worth value + 3 stat chips
- `TrendChart.jsx` - Recharts AreaChart with 30D/90D/1Y toggle
- `AllocationWidget.jsx` - Recharts PieChart donut + legend list
- `HoldingsWidget.jsx` - list with sparklines, top 8 by value
- `MoversWidget.jsx` - top 5 by absolute daily % change, horizontal bars
- `WatchlistWidget.jsx` - top 5 from watchlist with price/change

**Stock drawer** (`components/StockDrawer.jsx`):
- Fixed right overlay, 300px wide, `border-left: 1px solid #10b981`
- Quote + badge, price chart with range toggle, your position, key stats
- Backdrop click closes it; opens from any ticker click across all tabs

**State in `app/(dashboard)/page.jsx`:**
- `activeTab` - 'overview' | 'accounts' | 'portfolio' | 'watchlist'
- `drawerStock` - null | ticker string
- `netWorthData` - { value, change, changePositive } fed by OverviewTab via `onDataLoaded` callback

## Design System

BatchFolio uses a data-terminal aesthetic. Key rules:

**Colors (hardcode these hex values, not Tailwind color names):**
- Background: `#0d1117`
- Surface/card: `#161b22`
- Border: `#21262d`
- Accent/emerald: `#10b981` (hover: `#34d399`)
- Positive: `#34d399` | Negative: `#f87171`
- Text primary: `#e6edf3` | Text muted: `#7d8590`

See `lib/colors.js` for the authoritative design token reference.

**No purple, no violet, no blue anywhere.** BatchFolio uses emerald only.

**Typography:** `font-mono` on ALL ticker symbols, prices, share counts, percentages, and any numeric values. Inter (`font-sans`) for all labels, headings, nav text.

**Logo pattern:** `<span class="text-[#7d8590]">Batch</span><span class="text-[#10b981]">Folio</span>`

**Active nav:** 2px left border `border-l-[#10b981]` + `bg-[rgba(16,185,129,0.06)]`. Not a pill. Use `border-l-2 border-l-transparent` on inactive items to avoid layout shift.

**Cards:** `rounded-md` (not `rounded-xl`), `bg-[#161b22]`, `border border-[#21262d]`. Net Worth card uses `border-[#10b981]/40`.

**Tables:** Compact row padding `py-2.5`. Use `TableCell` from `components/ui/table.jsx`.

**Loading states:** Always use `<Skeleton>` from `components/ui/skeleton.jsx`, never a spinner or blank space.

**Errors:** Always show inline text with `text-[#f87171]`, never swallow errors silently.

**Dark theme only.** No light mode support. The `<html>` tag has `class="dark"`.

**No em dashes anywhere in the codebase.** Use hyphens or restructure the sentence.

**Shared utilities:**
- `lib/format.js` - `fmt()`, `fmtCompact()`, `fmtPercent()`, `fmtShares()` for all monetary/numeric formatting. Use these everywhere; do not inline number formatting.
- `lib/constants.js` - `DEMO_EMAIL` and other shared constants
- `lib/colors.js` - design token reference
- `lib/utils.js` - `cn()` (clsx + tailwind-merge)
- `SLICE_COLORS` array in `AllocationWidget.jsx` / `AllocationChart.jsx` for pie chart slices
- `ErrorBoundary` wraps all dashboard widgets to isolate failures

## Supabase Schema

All tables are in the `batchfolio` schema. Queries must use `.schema('batchfolio')` or the client must be initialized with `{ db: { schema: 'batchfolio' } }`.

```
accounts:
  id, user_id, name, type ('brokerage'|'retirement'), provider, created_at,
  is_synced, is_excluded, is_hidden, sort_order, balance, available_balance,
  currency, last_balance_date, simplefin_id, provider_url

holdings:
  id, account_id (cascade delete), ticker, shares, avg_cost_basis, updated_at,
  is_synced, last_synced_price, description, cost_basis_total, currency,
  purchase_price, simplefin_id

liabilities:
  id, user_id, name, type, balance, interest_rate, created_at,
  is_synced, simplefin_id

transactions:
  id, account_id, user_id, simplefin_id, posted_at, transacted_at,
  amount, description, payee, memo, pending, tx_type, liability_id

simplefin_connections:
  id, user_id, access_url, last_synced_at

net_worth_snapshots:
  id, user_id, date, total_assets, total_liabilities, net_worth

account_snapshots:
  id, user_id, account_id, balance, date. Unique on (account_id, date).
  Tracks daily account balance for all account types including bank and HSA accounts.

watchlist:
  id, user_id, ticker, added_at. Unique on (user_id, ticker).
```

Holdings RLS joins to accounts to verify `accounts.user_id = auth.uid()`. From the client, a plain `.from('holdings').select('*')` returns only the current user's holdings.

## API Routes

**Stock data (server-side proxies):**
- `GET /api/stock/quote?ticker=` - returns price, change, changePercent, high, low, open, previousClose, name (Finnhub)
- `GET /api/stock/fundamentals?ticker=` - returns marketCap, peRatio, eps, high52w, low52w, dividendYield, beta (Finnhub)
- `GET /api/stock/chart?ticker=&range=30d` - ranges: 30d, 90d, 1y - returns `{ candles: [{date, open, high, low, close, volume}] }` (Polygon.io)
- `GET /api/stock/sparkline?ticker=` - short candle series for sparklines (Polygon.io)
- `GET /api/stock/news?ticker=` - recent news articles (Finnhub)
- `GET /api/stock/search?q=` - symbol search/autocomplete (Finnhub)

**SimpleFIN:**
- `POST /api/simplefin/connect` - claim setup token, store access_url
- `POST /api/simplefin/sync` - pull latest balances and transactions
- `POST /api/simplefin/disconnect` - remove access_url

**Admin:**
- `POST /api/admin/send-invite`
- `GET /api/admin/list-invites`
- `POST /api/admin/revoke-invite`

**Other:**
- `GET /api/version` - returns build hash for auto-refresh on redeploy

Finnhub free tier: 60 API calls/minute. Polygon.io key goes in the `Authorization` header (`Bearer <key>`), not in the URL query string. Batch fetches use `Promise.all()`.

## SimpleFIN Integration

SimpleFIN syncs real bank/brokerage balances and transactions via a user-provided setup token. Flow:

1. User pastes a setup token into the UI
2. `/api/simplefin/connect` claims it to get a permanent `access_url`, stores it in `simplefin_connections`
3. `/api/simplefin/sync` fetches accounts and transactions using the stored `access_url`
4. Synced accounts set `is_synced = true` and are matched by `simplefin_id`
5. Credit card accounts are routed to `liabilities`; shadow accounts hold their transactions
6. The `access_url` is stored server-side only and never sent to the client

**Mutual funds from SimpleFIN:** use `last_synced_price` from the holdings row instead of calling Finnhub. Never call Finnhub for a holding where `is_synced = true` and `last_synced_price` is set.

## Edge Functions

**`nightly-snapshot`** (Supabase Edge Function):
- Triggered nightly at 23:00 UTC via `pg_cron` (cron: `0 23 * * *`)
- Syncs SimpleFIN first, then calculates and inserts a net worth snapshot for each user
- Skips the demo user (checked against `DEMO_EMAIL`)
- All `.from()` calls must use `.schema('batchfolio')` because the client is initialized without a default schema
- Secrets use `APP_SUPABASE_URL` and `SERVICE_ROLE_KEY` naming - the `SUPABASE_` prefix is reserved by Supabase and cannot be used for custom secrets

## Key Conventions

- **Demo user guard:** check `user.email === DEMO_EMAIL` (from `lib/constants.js`) before any write operation. Block writes silently or show a toast - never mutate demo data.
- **Ticker validation:** apply `/^[A-Z0-9.\-]{1,10}$/` before any external API call.
- **CASH ticker:** never open the stock drawer, never fetch a quote. Use `avg_cost_basis` as the price directly.
- **Synced mutual funds:** use `last_synced_price`; never call Finnhub for holdings where `is_synced = true` and a price is already stored.
- **Polygon API key:** pass in `Authorization: Bearer <key>` header, not as a URL query param.
- **All monetary values** use `fmt()` from `lib/format.js`. No inline `toLocaleString` or manual `$` concatenation.
- **No em dashes** anywhere in the codebase. Use hyphens or restructure the sentence.
