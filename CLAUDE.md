## Architecture

Next.js 15 App Router, JavaScript (no TypeScript), Tailwind CSS, shadcn/ui components, Supabase auth + database, Recharts for charts. Deployed on Vercel at batchfolio.batch-apps.com.

**Route groups:**
- `app/(auth)/` - login and signup pages, no sidebar/nav
- `app/(dashboard)/` - all protected pages, wrapped in `layout.jsx` with `Sidebar` + `BottomNav`
- `app/api/stock/` - server-side Finnhub proxy routes (quote, fundamentals, chart)

**Auth:** Middleware at `middleware.js` handles all redirects. Unauthenticated users hitting dashboard routes go to `/login`; authenticated users hitting auth pages go to `/`. All dashboard pages are `'use client'` and use `@/lib/supabase` (browser client) for data operations.

**Finnhub data is server-side only.** All stock data must go through `/api/stock/*` routes. Never call Finnhub from client components. `FINNHUB_API_KEY` is never exposed to the browser.

**Supabase clients:**
- `lib/supabase.js` - browser client (`createBrowserClient` from `@supabase/ssr`), used in client components
- `lib/supabase-server.js` - server client (`createServerClient`), used in server components and middleware
- RLS is enabled on all tables; queries automatically scope to `auth.uid()`

**shadcn components** live in `components/ui/`. All components are manually written (not generated by CLI) to match the dark theme. Do not override their dark styling.

**Shared components** in `components/`:
- `Sidebar.jsx` - desktop left nav (240px, fixed)
- `BottomNav.jsx` - mobile fixed bottom nav
- `StatCard.jsx` - metric card (label + monospace currency value)
- `NetWorthChart.jsx` - Recharts AreaChart for net_worth_snapshots
- `AllocationChart.jsx` - Recharts PieChart donut for portfolio allocation
- `StockPriceChart.jsx` - Recharts AreaChart for stock candle data

## Design System

BatchFolio uses a data-terminal aesthetic distinct from BatchFlow. Key rules:

**Colors (hardcode these hex values, not Tailwind color names):**
- Background: `#0d1117`
- Surface/card: `#161b22`
- Border: `#21262d`
- Accent/emerald: `#10b981` (hover: `#34d399`)
- Positive: `#34d399` | Negative: `#f87171`
- Text primary: `#e6edf3` | Text muted: `#7d8590`

**No purple, no violet, no blue anywhere.** BatchFolio uses emerald only.

**Typography:** `font-mono` on ALL ticker symbols, prices, share counts, percentages, and any numeric values. Inter (`font-sans`) for all labels, headings, nav text.

**Logo pattern:** `<span class="text-[#7d8590]">Batch</span><span class="text-[#10b981]">Folio</span>`

**Active nav:** 2px left border `border-l-[#10b981]` + `bg-[rgba(16,185,129,0.06)]`. Not a pill. Use `border-l-2 border-l-transparent` on inactive items to avoid layout shift.

**Cards:** `rounded-md` (not `rounded-xl`), `bg-[#161b22]`, `border border-[#21262d]`. Net Worth card uses `border-[#10b981]/40`.

**Tables:** Compact row padding `py-2.5`. Use `TableCell` from `components/ui/table.jsx`.

**Loading states:** Always use `<Skeleton>` from `components/ui/skeleton.jsx`, never a spinner or blank space.

**Errors:** Always show inline text with `text-[#f87171]`, never swallow errors silently.

**Dark theme only.** No light mode support. The `<html>` tag has `class="dark"`.

**No em dashes anywhere in the codebase.** Use hyphens or restructure the sentence.

## Supabase Schema

```
accounts:          id, user_id, name, type ('brokerage'|'retirement'), provider, created_at
holdings:          id, account_id (cascade delete), ticker, shares, avg_cost_basis, updated_at
liabilities:       id, user_id, name, type, balance, interest_rate, created_at
net_worth_snapshots: id, user_id, date, total_assets, total_liabilities, net_worth
watchlist:         id, user_id, ticker, added_at. Unique on (user_id, ticker).
```

Holdings RLS joins to accounts to verify `accounts.user_id = auth.uid()`. From the client, a plain `supabase.from('holdings').select('*')` returns only the current user's holdings.

## API Routes

- `GET /api/stock/quote?ticker=` - returns price, change, changePercent, high, low, open, previousClose, name
- `GET /api/stock/fundamentals?ticker=` - returns marketCap, peRatio, eps, high52w, low52w, dividendYield, beta
- `GET /api/stock/chart?ticker=&range=30d` - ranges: 30d, 90d, 1y - returns `{ candles: [{date, open, high, low, close, volume}] }`

Finnhub free tier: 60 API calls/minute. Batch fetches use `Promise.all()`.
